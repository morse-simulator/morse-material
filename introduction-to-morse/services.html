Kind of communications in robotic architectures

  • data flow or data streams exchanged between the components (continuous)
  • discrete requests (RPC) (configure components, request computations, ...)
    • services, actions for ROS
    • methods, commands for Orocos
    • services for OpenRTM
    • ...
    • services for Morse!

A simple example

from morse.builder import *

robot = ATRV()
robot.translate(z=0.2)

# Waypoint actuator
waypoint = Waypoint()
robot.append(waypoint)

waypoint.add_service('socket')


# A position sensor
pose = Pose()
robot.append(pose)
pose.add_service('socket')


env = Environment('outdoors')

Configuring services at the builder level

  • Configure your component with add_service instead of add_stream .
  • You can call these two methods if you want to have both services and datastreams for a component.

Controlling a component using socket

The pymorse approach

from pymorse import Morse
from time import sleep

with Morse() as morse:
    waypoint = morse.robot.waypoint
    pose = morse.robot.pose


    print("position %s" % pose.get_local_data().result())

    go = waypoint.goto(10.0, 0.0, 0.0, 1.0, 1.0)
    for i in range(0, 3):
        print("Position %s status %s\n" % (
                pose.get_local_data(), waypoint.get_status()))
        sleep(1.0)
    go.result()
    print("Position %s status %s\n" % (
                pose.get_local_data(), waypoint.get_status()))
