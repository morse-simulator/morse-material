What is MORSE?

The Modular OpenRobots Simulation Engine

Pierrick Koch
Apr, 2013

The MORSE project

  • 100% open-source, permissive BSD-like license
  • Initiated at LAAS-CNRS in 2009
  • As of Apr 2013, contributions by 10 institutions worldwide
  • 15 different contributors for the last release
  • ~20K LOC, mostly Python
  • primary a 3D modeller
  • provide a 3D game engine, support shaders
  • Switch from the modeller to the simulation == press 'P'
  • BGE: written in C++, extensible in Python + logic bricks

Modularity

  • MORSE is the Modular OpenRobots Simulation Engine
  • New components can be easily added (robots, sensors, actuators)
  • New middleware support as well

Interfaces

  • Generic: sockets, text,
  • Specific to robotics: middlewares
    • GenoM/pocolibs
    • MOOS
    • OpenRTM
  • Two main interaction types: RPC-oriented or datastream-oriented
  • Language bindings available for Python. Easy to develop for other languages through the socket interface.
This set of 4 slides show how MORSE can be used at different level of abstraction: from 'few things simulated, lot of robot software running' to 'almost everything simulated, only high-level robot softwares'.

Builder scripts

  • Allow to create a scene
  • Add components
  • Link them together
  • Link them to interfaces
  • Configure the simulation settings

Example builder

from morse.builder import *


# Add an ATRV robot
robot = ATRV()


# Add a motion actuator
motion = MotionVW()


# Add an odometry sensor
odom = Odometry()


# Parent actuator and sensor to our robot
robot.append(motion)
robot.append(odom)


# Set default interface
robot.add_default_interface('socket')


# Use an outdoor environment
env = Environment('outdoors')
