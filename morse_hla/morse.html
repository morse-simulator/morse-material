HLA : A general purpose architecture for distributed simulation systems

Introducing HLA

  • A standard defining
    • a way to describe a distributed simulation
    • interactions between runtime and simulators
    • rules that each simulator must comply with
  • Several simulators are HLA-compliant
  • Numerous RTI implementation exists (including a few open-source)

Describing a federation

  • Data Types
  • Objects : persistent thing over time
  • Interactions : non-persistent thing (discrete in time)
  • Modelling simulation interface is an hard problem !

Morse and HLA

  • Work developed at ISAE-SUPAERO, using CERTI implementation
  • Morse is now capable:
    • synchronise time with RTI (period-based)
    • synchronise "static" objects attributes
  • Missing points:
    • Interaction management
    • Really dynamic object
    • Data Distribution Management

Implementation details

  • HLA as a standard middleware
  • Main loop synchronizes (when needed) with RTIg
  • External objects as proxy object
  • Multi-nodes has been rewritten
